Generate Java Code From Wsdl

This topic describes the options available for fenerating Java Code From Wsdl.
  • Web service wsdl url - url location of webservice definition document, usually consists of url for the web service and additional suffix ?wsdl, e.g. http://127.0.0.1:8080/webservices/services/urn:simpleservice?wsdl
  • User name and Password - credentials for accessing wsdl url, should be set if wsdl url locations require authentication.
  • Output path - module source directory where generated java code stubs will be placed.
  • Package prefix - package for generated java files.
  • Generate TestCase - additional JUnit test case class will be generated for your testing purposes.
  • Web Service Platform - toolkit used for accessing/implementing the web service.
  • Binding to be used - some web service platform allow choice for xml to/from java mapping engine to be used.
  • Output mode - aside web service client code, some web service platforms allow for server side code generation.
  • Add required client libraries to classpath - generated stubs code depends on web service client libraries. If this option is set then they will be added to the classpath of the module that will contain the generated sources.
