Developing Web Services

This topic describes how to develop Web Services with IntelliJ IDEA.
  • Add a Web Facet to an existing Java module. Then configure it to contain the code.
      If you plan to deploy to JAX-WS2 in Java EE5 compatible implementation, the web application deployment descriptor version should be 2.5 or higher.
  • Add the Web Services Facet to the same Java module.
  • Run Enable Web Services Support action, select server type and specify web services engine to use, the action will update the web module with necessary libraries and servlet references
  • For web services enigne that does not support dynamic deployment (any other than Axis,e.g. JWSDP/Axis2/XFire etc) one needs to invoke Expose Class as Web Service action for existing class. The action will update web service descriptors and generate additional deployment code if needed, this action should be run once any web service method signature is changed, new method is added/removed, etc
  • Run / debug the web module with local Tomcat, having specified selected web services module to be deployed (in the Deployment tab of the run configuration)
  • For Axis one should deploy / undeploy the particular class to be web service dynamically. You can also see list of deployed web services.
  • It is possible to generate wsdl from existing java code.
  • One can monitor SOAP messages flow when developing with Axis (if appropriate option is specified during deployment).
  • Use your web service
  • Navigate from java code (web services client/service code class,method,field) to corresponding wsdl/wsdd elements.
  • Navigate within wsdl file, from binding attribute in port tag or from message attribute in input or output tags.
    Navigate from wsdl to java code, from name attribute of complexType,element, portType, operation inside port_type and binding, or from type attribute of binding or type to class or method implementing the service (interface for client) or stub class. Navigation from wsdl/java code created via other WebService toolkits (not Axis) may not work.
  • Navigate from wsdd to java code, from value attribute of parameter with allowedMethods name to method of the service class.
  • Use Ant tasks for automation of the web services related functionality. Necessary libs are in <webservices-plugin-dir>/lib. Here is Axis userguide, one also may need more info on Axis descriptors and runtime configuration, it is available here here. Documentation on using Axis with JMS is also available
  Web container JDK version should match JDK used for developing web services (otherwise Web Services code fails to load in server VM and one would get UnsupportedClassVersionError)
   If running web container produces unexpected Address already in use messages then one needs to shutdown orphaned web container instance outside of IntelliJ IDEA (e.g. running shutdown.bat for Tomcat)
   Certain web service engines experience difficulties when attempting to deploy Java classes in default package as web services.
