Deploying Web Services

This topic describes the options available for deploying Web Services.
  • Class for operation class to be used for deploy or undeploy
  • Name of the service name of the service for publishing (e.g. empty text for ROOT context or mycontext for /mycontext)
  • Web context name name of the server context that is used for running the web module containing the web service.
  • Service scope is scope of class instances during execution, new instance is created by runtime per request, session or application.
  • Methods for operations is table of available class methods, public class methods that have compatible return and parameter types for deploying the class as webservice are available for selection, noncompatible methods are greyed out, already deployed methods are bolded. Tooltip with current status is available on method name.
  • Enable SOAP messages monitoring allows monitoring SOAP messages via corresponding action.
  Note this prerequisite: Server should be launched for the Web Services Web Module.
