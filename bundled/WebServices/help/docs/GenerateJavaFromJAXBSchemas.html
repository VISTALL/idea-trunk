Generate Java Code From Xml Schema using JAXB

This topic describes the options available for generating Java Code From Xml Schema using JAXB.
  • JAXB schemas / wsdl / dtd path - file to be used for generation.
  • Output path - module source directory where generated java code stubs will be placed.
  • Package prefix - package for generated java files.
  • Generate package level annotations - when on, package-info.java containing annotations code is generated. When off - those annotations internalized into the other generated classes.
  • Mark generated code with 'generated' annotation - Mark the code with javax.annotation.Generated annotation.
  • Make generated file read-only - use this option to force the XJC binding compiler to mark the generated Java sources read-only.
  • Add required client libraries to classpath - generated stubs code depends on additional JAXB client libraries. If this option is set then they will be added to the classpath of the module that will contain the generated sources.
