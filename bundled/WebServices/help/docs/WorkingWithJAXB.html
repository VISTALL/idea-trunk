Working with JAXB

This topic describes how to work with JAXB.
  • Generate JAXB schemas from existing code (Tools | JAXB | Generate JAXB Schemas From Java).
  • Generate java from JAXB schemas (Tools | JAXB | Generate Java from JAXB Schemas).
  • It is possible to generate marshalling / unmarshalling JAXB code by invoking Code | Generate action. The alternative way of such generation is to use live templates mjo (marshal java object) and ujo (unmarshal java object) to convert JAXB mapped objects to xml and back.
  • Set JWSDP path in Settings | WebServices
  • Navigate from JAXB mapped java code back to JAXB schema (override markers)
  • Navigate from JAXB schema to java code (name of complexType or element).
  • Plugin validates JAXB mapped java code in order to match JAXB requirements (e.g. Java bean conventions for exposed classes/properties).
